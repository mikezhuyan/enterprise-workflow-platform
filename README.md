# Enterprise Workflow Platform

企业级智能工作流平台 - 融合传统工作流与AI能力的下一代业务流程编排系统。

## 🌟 核心特性

- **🔧 可视化工作流设计器**：拖拽式节点编排，支持复杂业务流程
- **📦 组件注册中心**：企业内部技术组件统一管理、版本控制、发布订阅
- **🤖 MCP 集成**：支持 Model Context Protocol，与大模型无缝集成
- **🎭 Agent Skills**：可扩展的智能体技能系统
- **🔌 企业级协议支持**：REST/gRPC/GraphQL/消息队列等传统协议
- **👥 完善权限体系**：RBAC 权限控制，支持多租户
- **⚡ 高可用架构**：支持分布式部署和水平扩展

## 🚀 快速开始

### 环境要求

- Docker 20.10+
- Docker Compose 2.0+
- Node.js 20+ (本地开发)
- Python 3.11+ (本地开发)

### 使用 Docker Compose 启动

```bash
# 进入项目目录
cd enterprise-workflow-platform

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

访问:
- 前端界面: http://localhost:8080
- API文档: http://localhost:8080/api/docs

### 默认账号

- 用户名: `admin`
- 密码: `admin123`

### 本地开发

#### 后端开发

```bash
cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 初始化数据库
python scripts/init_data.py

# 启动开发服务器
uvicorn app.main:app --reload
```

后端服务将在 http://localhost:8000 运行

#### 前端开发

```bash
cd frontend

# 安装依赖 (使用 pnpm)
pnpm install

# 启动开发服务器
pnpm dev
```

前端服务将在 http://localhost:5173 运行

## 🔌 支持的组件类型

### 传统组件
- **API组件**: HTTP/HTTPS/gRPC调用
- **数据库组件**: SQL/NoSQL操作
- **消息组件**: MQ/Kafka/RabbitMQ
- **脚本组件**: Python/JavaScript执行
- **控制组件**: 条件判断、循环、延时、子工作流

### AI组件
- **LLM组件**: GPT-4/Claude等大模型调用
- **MCP组件**: Model Context Protocol工具调用
- **Agent组件**: 智能体执行和技能调用
